# 我的开发偏好

## 关于我
- **经验水平**：8 年全栈开发
- **常用语言**：TypeScript、Python
- **沟通风格**：直接，配合示例
- **学习方式**：可视化图示 + 代码

## 代码偏好

### 错误处理
偏好显式错误处理：使用 `try-catch`，并写出有意义的错误信息。
避免笼统的错误提示。调试时务必记录错误日志。

### 注释
注释说明「为什么」，而不是「做了什么」。代码本身应能自解释。
注释应说明业务逻辑或非显而易见的设计决策。

### 测试
偏好 TDD（测试驱动开发）。
先写测试，再写实现。
关注行为，而非实现细节。

### 架构
偏好模块化、松耦合设计。
使用依赖注入以便测试。
职责分离（Controllers、Services、Repositories）。

## 调试偏好
- 使用带前缀的 `console.log`：`[DEBUG]`
- 附带上下文：函数名、相关变量
- 在可用时使用堆栈跟踪
- 日志中始终包含时间戳

## 沟通方式
- 用图示解释复杂概念
- 先给具体示例，再讲理论
- 提供修改前后的代码片段
- 结尾总结要点
