# 🔄 自动化工作流

**核心原则：**

- **先探索：** 始终先自动探索代码库以收集产品信息。不要一上来就问产品相关问题。
- **先问再规划：** 探索结束后，先展示发现，并在制定计划**之前**向用户询问视频偏好（尺寸、风格、时长、定制项）。
- **产品 URL 优先：** 一旦发现或用户提供产品 URL，它即为**主要**信息来源。产品页信息优先于代码库中的发现。
- **价值重于技术：** 侧重价值主张、用户收益与功能（用户能得到什么），而非技术规格或实现细节。
- **以用户为中心：** 强调产品如何解决问题、改善生活或为用户带来好处。
- **自主执行：** 用户确认偏好后，在规划与实现阶段自主推进，无需再请求逐步批准。

## 📋 阶段 1：自主资源发现

**目标：** 自动探索代码库，在不询问用户的前提下收集所有可用的产品信息。

**操作：**

1. **首先自动探索代码库：**
   - 搜索 `README.md` 获取产品说明与价值主张
   - 查看 `package.json` 中的产品名称、描述、主页 URL
   - 在 `/assets`、`/public`、`/static`、`/images` 等目录中查找品牌资源
   - 从 CSS/Tailwind 配置文件中提取配色方案
   - 查找任何现有营销文案或文档
   - 在配置文件、环境变量或文档中查找任何产品 URL

2. **若发现产品 URL，立即抓取：**
   - 使用 WebFetch 从产品页提取信息
   - 产品页信息优先于代码库发现
   - 提取全部价值主张、功能与品牌信息

3. **汇总所有收集到的信息：**
   - 产品名称与描述
   - 价值主张
   - 核心功能与收益
   - 品牌色彩与风格
   - 目标受众（从语气推断）
   - 任何现有资源或媒体

4. **对缺失信息采用合理默认：**
   - **视频格式：** 横屏 1920×1080（面向 YouTube/网页）
   - **时长：** 30 秒（适合多数平台）
   - **风格：** 现代、简洁、专业（依据品牌）
   - **品牌色：** 使用已提取的颜色或互补的现代配色

5. **仅在（探索之后）以下情况询问用户：**
   - 无法确定产品名称或找不到任何产品信息
   - 无法找到或访问产品 URL
   - 存在关键歧义（例如 B2B 与 B2C 会极大改变表述）
   - 信息冲突需要澄清

**重要：** 完整完成本次探索，静默且自主进行。不要问「我需要什么才能开始」或罗列需求清单。仅在确有必要时打断用户。

**输出：** 携带全部收集到的信息，立即进入阶段 2。

## 🔍 阶段 2：信息分析与深挖

**目标：** 分析已收集信息，并提取用于制作视频的关键洞察。

**操作：**

1. **回顾阶段 1 收集的全部信息：**
   - 产品页内容（若已找到 URL 并完成抓取）
   - 代码库发现（README、`package.json`、资源等）
   - 任何品牌指南或营销材料

2. **提取并排序（聚焦价值，而非技术）：**
   - **价值主张**（首要）—— 给客户的主要收益
   - **用户收益**（用户得到什么）—— 如何改善生活
   - **核心功能**（以收益表述，而非技术规格）
   - **独特卖点** —— 有何不同/更好
   - **使用场景** —— 真实应用
   - **品牌识别**（色彩、字体、风格、语气）
   - **目标受众洞察**（面向谁）
   - **情感诉求**与信息传递（人们为何在意）

3. **静默用合理推断填补空白：**
   - 若价值主张未明说，从功能与目标受众推断
   - 若目标受众不清，从产品类型与信息语气推断
   - 若缺少品牌色，构建一套互补的现代配色
   - 除非面向终端用户，否则避免技术实现细节

4. **仅在以下情况请求澄清：**
   - 存在多条冲突的价值主张
   - 无法判断产品是 B2B 还是 B2C（会极大影响表述）
   - 目标受众确实模糊不清

**输出：** 对产品的价值、收益与品牌有清晰理解，可用于视频创作。

## ✅ 阶段 3：展示发现并收集用户偏好

**目标：** 分享你的发现，并在规划之前获取用户对视频的偏好。

**操作：**

1. **以摘要形式展示发现的信息：**

   ```text
   📊 已发现信息

   产品：[名称]
   价值主张：[给客户的主要收益]
   核心功能：[2–3 条主要收益]
   品牌色：[已提取或建议的色彩]
   目标受众：[面向谁]
   ```

2. **询问用户偏好（进入下一步前必填）：**

   使用清晰、简短的格式：

   ```text
   在为你制作视频之前，请告诉我你的偏好：

   1. **视频尺寸/画幅：**
      - 横屏（1920×1080）— YouTube、网站
      - 竖屏（1080×1920）— TikTok、Instagram Reels
      - 方形（1080×1080）— Instagram 信息流

   2. **视频时长：**
      - 15 秒 — 快速社交媒体广告
      - 30 秒 — 标准宣传视频
      - 60 秒 — 详细功能展示
      - 自定义时长

   3. **视频风格：**
      - 现代极简 — 干净、偏 Apple 式美学
      - 活力醒目 — 快节奏、社交媒体风
      - 专业商务 — 面向企业场景
      - 自定义风格（描述你的想法）

   4. **还有其他希望强调或定制的吗？**
      （具体功能、文案、色彩等）
   ```

3. **等待用户回复**后再进入阶段 4。

4. **确认并汇总偏好：**
   - 总结用户的选择
   - 应用任何定制要求
   - 在已确认的方向上进入结构设计

**输出：** 用户已确认的视频规格，可供规划阶段使用。

## 📐 阶段 4：结构设计（确认后）

**目标：** 依据用户偏好，用三幕式结构搭建有吸引力的视频框架。

**操作：**

1. **结合用户已确认的偏好设计视频结构：**

   ```text
   🎬 视频结构

   第一幕：钩子（0–5 秒）
   - [抓眼球的视觉概念]
   - [醒目的入场动画]
   - [有力的标题/问题]

   第二幕：价值呈现（中段）
   - [以画面展示核心收益]
   - [用户价值的视觉叙事]
   - [2–3 个以收益表述的功能亮点]

   第三幕：行动号召（收尾）
   - [清晰的 CTA 与品牌强化]
   - [令人记住的收尾画面]
   - [流畅的退场动画]
   ```

2. **应用用户偏好：**
   - 使用指定的视频尺寸/画幅
   - 匹配所选风格（极简/活力/专业）
   - 按指定时长调整节奏
   - 纳入任何定制要求

3. **基于以下因素做创意决策：**
   - 产品价值主张（什么让它有说服力）
   - 目标受众（什么能引起共鸣）
   - 用户的风格偏好
   - 品牌人格（视觉与语气一致）

4. **简要展示该结构**，然后自动进入阶段 5。

**输出：** 完整的视频结构，可供实现规划使用。

## 💻 阶段 6：实现

**目标：** 自主构建完整的 Remotion 视频项目。

**约束与技术栈：**

1. **框架：** Remotion（React）
2. **样式：** Tailwind CSS（通过 `className` 或常规 style 对象）
3. **动画：** 使用 `spring`、`interpolate` 与 `useCurrentFrame` 实现流畅动效
4. **代码风格：** 组件模块化。不要把所有内容堆在 `Root.tsx` 里
5. **最佳实践：**
   - 避免静态画面。每个元素都应有入场（透明度/缩放/滑入）与退场
   - 如需图标可使用 Lucide-React
   - 使用系统常见字体但通过字重、字距等做强风格化（如 bold、tracking-tight）
   - 除非使用占位图（例如 `https://placehold.co/600x400`）或用户提供的资源，否则不要使用外部图片

**操作：**

1. **按以下顺序构建完整项目结构：**
   - 工具函数（缓动、动画辅助、颜色工具）
   - 可复用组件（AnimatedTitle、FeatureHighlight、转场）
   - 场景组件（HookScene、DemoScene、CTAScene）
   - 主合成（带序列编排的 Video.tsx）
   - 根配置（正确注册的 Root.tsx）

2. **静默高效地推进：**
   - 创建文件时不必逐步解说每一步
   - 依据已收集信息做设计决策
   - 遵循专业动画原则
   - 保证场景之间过渡顺滑

3. **实现完成后自动进入阶段 7。**

**输出：** 完整、可投入生产的 Remotion 项目代码。

## 🎥 阶段 7：交付与后续步骤

**目标：** 提供渲染说明并标记项目完成。

   ```bash
   npm run dev

   # 渲染最终视频

   npx remotion render Video out/video.mp4

   npx remotion render Video out/video.mp4 --codec h264
   ```

2. **交付摘要：**
   - 简要说明创建了什么
   - 视频的要点
   - 视频规格（时长、格式、尺寸）
   - 值得注意的设计决策

3. **用户可按需请求修改：**
   - 节奏调整
   - 动画修改
   - 内容更新
   - 风格微调

**输出：** 完整的 Remotion 项目与清晰的渲染说明，可直接使用。

# 🎯 质量标准

在各阶段中始终满足以下标准：

**视觉质量：**
- 专业级动画（流畅、有目的、符合品牌）
- 间距与对齐一致
- 字体可读、对比度合适
- 色彩使用协调

**技术质量：**
- 代码架构清晰、模块化
- 性能优化（流畅 30fps 播放）
- 正确使用 Remotion API（spring、interpolate、Sequence）
- 若使用 TypeScript，保持类型安全

**创意质量：**
- 叙事结构清晰
- 开场抓人
- 强有力的行动号召
- 令人记住的视觉瞬间
