<a id="contributor-code-of-conduct"></a>
# 贡献者行为准则

<a id="our-commitment"></a>
## 我们的承诺

我们致力于为所有人营造友好且富有启发性的社区。我们承诺，参与本社区对每个人而言都应是免受骚扰的体验，无论其年龄、体型、种姓、残障状况、族裔、性征、性别认同与表达、经验水平、教育程度、社会经济地位、国籍、外貌、种族、宗教或性取向如何。

我们致力于提供积极、包容且安全的环境，使所有贡献者感到受尊重、受重视，并能够尽力贡献。

<a id="our-standards"></a>
## 我们的标准

有助于营造积极环境的行为示例包括：

<a id="be-respectful"></a>
### 相互尊重
- 使用友好且包容的语言
- 尊重不同的观点、立场与经历
- 虚心接受建设性批评
- 尊重文化与语言差异
- 承认并欣赏多元背景

<a id="be-collaborative"></a>
### 协作共进
- 协力化解冲突
- 给予认可并肯定他人的贡献
- 帮助他人学习与成长
- 分享知识并提出问题
- 支持社区中的其他成员

<a id="be-professional"></a>
### 保持专业
- 使讨论聚焦且富有成效
- 避免贬低或侮辱性言论
- 在议题与讨论中紧扣主题
- 使用清晰且尊重的沟通方式
- 在解读他人信息时善意推定

<a id="be-inclusive"></a>
### 促进包容
- 欢迎新贡献者
- 帮助刚接触本项目的人上手
- 提供建设性反馈
- 纳入不同视角的参与者
- 为较少发声者留出表达空间

<a id="unacceptable-behavior"></a>
## 不可接受的行为

下列行为视为骚扰，在本社区中不可接受：

<a id="harassment-and-discrimination"></a>
### 骚扰与歧视
- 与性别、性别认同与表达、性取向、残障、精神疾病、神经多样性类型、外貌、体型、年龄、种族或宗教相关的冒犯性评论
- 对他人生活方式与选择的不当评论
- 故意错用性别称谓，或使用对方已弃用或拒绝使用的姓名
- 无关或不当的性意味图像或行为
- 未经同意或在对方要求停止后的身体接触或模拟身体接触

<a id="abusive-behavior"></a>
### 虐待性行为
- 针对他人的暴力威胁或暴力性言辞
- 煽动暴力或对任何个人使用暴力性言辞
- 故意恐吓
- 跟踪或尾随
- 通过反复联系实施骚扰
- 其他在合理情形下可视为不当的行为

<a id="online-harassment"></a>
### 网络骚扰
- 持续扰乱线上讨论
- 公开他人隐私信息（人肉搜索）
- 不受欢迎的性关注或性示好
- 引战或故意扰乱秩序
- 垃圾信息或重复广告
- 传播不实信息或误导性信息

<a id="retaliation"></a>
### 报复
- 因他人举报违规行为而采取报复行动
- 因他人举报不当行为而对其不公对待
- 以举报相威胁作为报复

<a id="scope"></a>
## 适用范围

本行为准则适用于：
- 所有社区空间（GitHub issues、pull requests、discussions 等）
- 项目仓库与文档
- 由本项目组织或代表本项目举办的活动与聚会
- 个人代表本项目或本社区的公开场合
- 与项目事务相关的私人通信

本行为准则适用于所有项目相关空间；当个人在公开场合代表本项目或本社区时，同样适用。代表项目或社区的示例包括：使用官方项目电子邮箱、通过官方社交媒体账号发帖，或在线上/线下活动中担任指定代表。

<a id="enforcement"></a>
## 执行

社区负责人负责阐明并执行可接受行为标准，并对任何其认为不当、威胁性、冒犯性或有害的行为采取适当且公正的纠正措施。

<a id="reporting-a-violation"></a>
### 如何举报

若您遭遇或目睹不可接受的行为，或有其他关切，请通过以下方式举报：

1. **电子邮件**：联系项目维护者，[如有可用联系方式请在此填写]
2. **GitHub Issues**：私密举报功能（若您的 GitHub 设置中提供）
3. **私信**：直接联系项目维护者

请尽量提供：
- 您的联系方式
- 涉事者姓名（若您愿意分享）
- 事件说明
- 背景与相关链接
- 事件对您的影响

**所有举报均将保密处理。**

<a id="investigation-and-response"></a>
### 调查与回应

社区负责人将：
- 及时、公正地审查并调查所有投诉
- 对举报人的信息予以保密
- 向举报人反馈处理进度
- 根据调查结论采取适当措施

可能采取的措施包括：
- 私下警告或辅导
- 公开道歉（若违规者同意）
- 暂时或永久移出社区空间
- 取消贡献者权限
- 必要时向 GitHub 或其他平台升级处理

<a id="appeal-process"></a>
### 申诉程序

若您认为决定有误，可通过以下方式申诉：
1. 提供新的信息或背景
2. 请求由多名维护者复核
3. 在初次决定作出后 30 日内跟进

<a id="guidelines-for-respectful-disagreement"></a>
## 尊重性分歧指南

技术社区中出现分歧很自然。以下是如何在尊重前提下表达不同意见：

<a id="dos"></a>
### 宜做之事
- ✅ 就事论事，不针对个人
- ✅ 提出澄清性问题
- ✅ 承认对方观点中的合理之处
- ✅ 使用「我」开头的表述（「我不同意，因为……」）
- ✅ 引用来源与证据
- ✅ 感谢他人分享观点

<a id="donts"></a>
### 不宜之事
- ❌ 人身攻击
- ❌ 使用轻蔑措辞（如「这太蠢了」）
- ❌ 恶意推定
- ❌ 翻出无关的旧怨
- ❌ 抬高声调或变得敌对
- ❌ 故意歪曲他人观点

<a id="example-of-respectful-disagreement"></a>
### 尊重性分歧示例
```
❌ 不当：「这主意太蠢了，你根本不懂这个项目。」

✅ 恰当：「我理解你的出发点，但我担心这种方式在更大项目上
可能难以扩展。你是否考虑过这份基准测试里体现的性能影响？
我很想听听你在这方面的想法。」
```

<a id="building-bridges"></a>
## 搭建桥梁

我们相信对话与和解的力量：

- 若有人指出您的行为，请倾听并反思
- 若造成伤害，愿意道歉
- 支持他人学习与成长
- 认识到人都会犯错
- 聚焦解决方案，而非归咎
- 在要求担责的同时善意推定

<a id="benefits-of-following-this-code"></a>
## 遵守本准则的益处

通过维护相互尊重且包容的社区，我们能够：
- 吸引背景多元的贡献者
- 通过协作得出更好的方案
- 建立更稳固的关系
- 使项目对新成员更加友好
- 产出更高质量的文档与示例
- 培育学习与成长的文化

<a id="special-considerations"></a>
## 特别说明

<a id="for-maintainers"></a>
### 面向维护者
- 以身作则遵守本准则
- 及时响应举报
- 决策过程保持透明
- 公正协助化解冲突
- 为所有人营造安全环境

<a id="for-contributors"></a>
### 面向贡献者
- 对维护者与其他贡献者善意推定
- 提供建设性反馈
- 支持新成员
- 协助维护社区标准
- 共同塑造社区文化

<a id="for-everyone"></a>
### 面向所有人
- 记住每条消息背后都是真实的人
- 若感到沮丧，不妨休息一下
- 如需支持，可联系社区负责人
- 共同建设您希望身处的社区
