```

Claude：我来为博客 API 制定一份完整计划。

**预计耗时**：4.5 小时  
**待创建文件**：约 25 个  
**主要技术**：Express、PostgreSQL、JWT、TypeScript

**接口汇总**：
- 认证：2 个
- 文章：6 个
- 评论：3 个

是否继续？（是/否/修改）
```

<a id="key-takeaways"></a>
## 要点总结

<a id="benefits-of-planning-mode"></a>
### 规划模式的好处

1. **清晰**：动手前有明确路线图
2. **估算**：时间与工作量预估
3. **风险评估**：尽早发现潜在问题
4. **优先级**：任务顺序合理
5. **确认**：执行前先审阅并批准
6. **可调整**：根据反馈修改计划

<a id="when-to-use-planning-mode"></a>
### 何时使用规划模式

✅ **适合使用**：
- 跨多日的项目
- 团队协作
- 关键系统变更
- 学习新概念
- 复杂重构

❌ **不必使用**：
- 修 bug
- 小改动
- 简单问答
- 快速试验

<a id="best-practices"></a>
### 最佳实践

1. 批准前**仔细审阅**计划
2. 发现问题时**修改计划**
3. **拆分**复杂任务
4. **估算**切合实际的时间
5. **纳入回滚**策略
6. **写明成功**标准
7. **每个阶段都规划测试**
