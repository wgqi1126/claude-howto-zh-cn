## Phase 1 - 基础设施与薄弱模块（2026 年 4 月）

**目标里程碑：M1**

### 支柱 4：新手引导

- [ ] **T-001** - 编写 `scripts/quickstart.sh`
  - 检测 `~/.claude/` 目录（如不存在则先向用户确认后创建）
  - 将第一个 slash command + `CLAUDE.md` + skill 复制到用户环境
  - 在 `CLAUDE.md` 中追加 agent-discovery 行，并链接到 `AGENT-INDEX.md`
  - 保证幂等：若文件已存在则提示并跳过，绝不覆盖
  - 完成后输出下一步操作建议

- [ ] **T-002** - 创建 `QUICKSTART.md`（前 15 分钟可视化指南）
  - 以带说明的终端步骤代码块呈现（尽量避免 PNG 截图，优先使用 ASCII 或 Mermaid）
  - 记录每张可视化内容对应的命令，便于后续重新生成

- [ ] **T-003** - 为 10 个模块的 README 全部补充难度徽章
  - 格式：`> 🟢 **Beginner** | ⏱ 30 minutes` 放在每个模块 README 顶部
  - 在徽章下方增加 “What you'll build” 预览 bullet

### 支柱 7：持续演进课程体系

- [ ] **T-004** - 创建 `WHATS-NEW.md`
  - 格式：`## CC vX.X - DATE`，每个版本下用 bullet 列出功能变化
  - 至少补上一条当前 CC 版本的记录

- [ ] **T-005** - 为 10 个模块 README 全部加入版本徽章
  - 格式：`> ✅ Verified against Claude Code **vX.X.X** · Last verified: YYYY-MM-DD`
  - 在每个模块 frontmatter 中加入 `cc_version_verified`

- [ ] **T-006** - 创建 `.github/workflows/staleness-check.yml`
  - 调度频率：每周一 09:00 UTC
  - 读取各模块 frontmatter 中的 `last_verified` 日期（使用 `yq` 或 10 行左右的 Python 脚本）
  - 对 30+ 天未验证的模块自动创建 GitHub Issue
  - 若已有同标题的 open issue，则跳过

### 支柱 6：内容质量（CLAUDE.md）

- [ ] **T-007** - 为项目创建根目录 `CLAUDE.md`
  - 作为项目自身配置来演示最佳实践
  - 记录贡献约定、模块结构与维护流程

### 支柱 2：AI Agent 索引（生成器）

- [ ] **T-008** - 编写 `scripts/build-agent-index.py`
  - 读取全部 10 个模块 README
  - 提取：标题（-> 主题）、代码块（-> 带语言标记的示例）、表格（-> 参考数据）
  - 输出 `agent-manifest.json`，其 schema 由 CEO 计划定义
  - 输出 `AGENT-INDEX.md`（面向人类阅读的摘要）
  - 校验引用是否指向真实存在的文件

### 模块 08-checkpoints：完整 deep pass（311 -> 800+ 行）

- [ ] **T-009** - 用真实场景重写模块开场
  - 示例：“你的重构刚好把 3 处地方弄坏了，这时 checkpoints 如何帮你……”

- [ ] **T-010** - 增加 checkpoint 策略决策树（Mermaid 流程图）
  - “我想做 X” -> 顺着箭头走 -> 落到对应 checkpoint 模式

- [ ] **T-011** - 增加 3 个工作流模板
  - 安全重构工作流
  - 带 checkpoints 的 A/B 测试
  - 灾难恢复工作流

- [ ] **T-012** - 增加与 planning mode 的联动说明

- [ ] **T-013** - 增加 checkpoint 生命周期 Mermaid 图

- [ ] **T-014** - 增加 2 个以上可验证的 “Try It Now” 区块
  - 每个区块包含：上下文说明、要执行的命令、预期输出说明

- [ ] **T-015** - 增加 “Patterns & Recipes” 区块，至少含 2 个命名模式
  - 每个模式包含：问题、解决方案代码、适用场景、不适用场景

### 模块 06-hooks：完整 deep pass

- [ ] **T-016** - 用真实场景重写模块开场
  - 示例：“你的 CI 在 20 分钟后才发现一个 lint 错误，而 hooks 可以在 2 秒内拦住它……”

- [ ] **T-017** - 增加 hooks 决策树（Mermaid 流程图）
  - “我想在 Y 发生时触发 X” -> 顺着箭头走 -> 落到对应 hook 类型

- [ ] **T-018** - 增加 5 个完整 hook 示例（每个完整脚本约 80 行）
  1. 写入时自动格式化（prettier/black）
  2. 提交时安全扫描（trufflehog/bandit）
  3. 文件变更时自动跑测试
  4. 会话结束时发送通知
  5. Prompt 校验（阻止危险模式）

- [ ] **T-019** - 增加 2 个以上可验证的 “Try It Now” 区块

- [ ] **T-020** - 增加 “Patterns & Recipes” 区块，至少含 2 个命名模式

**M1 检查点：基础设施上线，2 个最薄弱模块完成升级**
